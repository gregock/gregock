## Maintenance Rules

- Repository descriptions and README claims must match public evidence.
- Private runtimes, customer data, operational records, credentials, and private integrations must remain unpublished.
- Local folder names may differ from public repository names; `origin` is the authority for repository identity.

## Maintenance Workflow

1. Inspect current state.
2. Validate claims against the relevant repository.
3. Make minimal changes.
4. Review the diff.
5. Commit only after explicit approval.
6. Verify the resulting public GitHub state.
